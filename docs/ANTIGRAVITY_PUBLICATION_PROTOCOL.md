# Antigravity Publication Protocol

Status: active from 2026-05-24.

This protocol protects authorship, public history, and clean separation between open research and private laboratory work.

The broader cross-system development protocol is maintained in `docs/ANTIGRAVITY_DIRECTION_DEVELOPMENT_PROTOCOL.md`.

## 1. Draft

Drafts start in the private laboratory repository.

Drafts may include incomplete thinking, raw notes, open questions, and internal comments.

## 2. Review

Before public release, review the material for:

— secrets;
— private data;
— partner-sensitive data;
— unverified factual claims;
— unclear authorship;
— missing source links;
— missing license notice;
— whether the material is research text, code, data, or model output.

## 3. License Decision

For public non-code research materials, use:

Creative Commons Attribution 4.0 International (CC BY 4.0)

Official license:
https://creativecommons.org/licenses/by/4.0/

If the release contains code, simulation software, AI model files, or engineering systems, decide separately whether MIT, Apache 2.0, GPL, or another software license is required.

## 4. Public Commit

A public release must be committed to the relevant public repository with a clear commit message.

The public commit creates a timestamped version history.

## 5. Public Record

For important releases, create at least one public GitHub issue or discussion that records:

— release title;
— repository link;
— license;
— date;
— scope of materials;
— source files;
— any important caveats.

## 6. Citation

Public repositories must include `CITATION.cff` so GitHub can show citation information.
